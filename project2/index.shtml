Project 2

You can do this project alone, or with one other human in the class.

Write a Web app that tracks ticket sales at a zoo. Tickets for adults cost $10 each. Tickets for kids cost $6 each.

Client-side only. Use jQuery. Use store.js to persist the data.

After adding some data, the app might look like this:

The delete button deletes an entry.

The New button adds a new entry. Assume that the user does not make any mistakes. You can just use prompt(), if you want.

New entries don't have to appear in the right order, sorted by day.

When you add or remove an entry, the stats update automatically.

Add appropriate comments, as in the samples on this site.

At the top of the page, show the name(s) of the human(s) who did the project.

If you do the basics described so far, you will get 15 out of 20. For extra points:

  • When new data is added, show it in day order.
  • When the user clicks New, show a form at the bottom of the page, instead of using prompt(). Hide the form after the data is entered.
  • Add input validation. Numeric data only, no values less than zero.
  • Add an edit button to each entry. When the user clicks it, replace the data values with form fields. Hide the Delete and Edit buttons, and show Save and Cancel buttons instead. (2 points)

Submit the URL of your page in Moodle. List the extra things you did, if any. Type the name(s) of the human(s) who did the project.
